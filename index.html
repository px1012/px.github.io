此处报错的原因是因为:

显示

Hi px1012! You've successfully authenticated, but GitHub does not     provide shell   access.

就表明成功了

python安装过程问题解决方法

1.浅拷贝（copy）和深拷贝（deepcopy）

1）浅拷贝
只是拷贝了最外围对的对象本身，本身的内部元素都只是拷贝了一个引用。没有拷贝子对象，所以原始数据改变，子对象会改变。
2）深拷贝
deepcopy对外围和内部元素都进行了拷贝对象本身而不是对象的引用。原始对象的改变不会造成深拷贝里任何元素的改变

输出结果：

a =  [1, 2, 3, 4, ['a', 'b', 'c'], 5]
b =  [1, 2, 3, 4, ['a', 'b', 'c'], 5]
c =  [1, 2, 3, 4, ['a', 'b', 'c']]
d =  [1, 2, 3, 4, ['a', 'b']]

a是原始对象，作为对照；
b是a的引用，和a指向同一个对象；
c是a的一个浅拷贝，将a的内容拷贝过来了，就是和a平行的两个对象了，但是里面元素如果是象，仅拷贝其引用，即a[4]变化后c[4]也对应变化了，因为他俩实际指向同一个对象，而a和c指不同的对象；
d则是从头到尾全部拷贝过来，就连里面的对象也拷贝了一份，因此a[4]变化而d[4]没有变化，因为a[4]和d[4]在拷贝的时候值一样，但是其实已经是两个对象了。

2.迭代器和生成器

1）迭代器
迭代器是访问集合元素的一种方式。迭代器对象从集合的第一个元素开始访问，知道所有的元素被访问完结束。迭代器只能往前不会后退。
迭代器更大的功劳是提供了一个统一的访问集合的接口，只要定义了iter()方法对象，就可以使用迭代器访问。
迭代器有两个基本的方法
next方法：返回迭代器的下一个元素
iter方法：返回迭代器对象本身

2)生成器
带有 yield 的函数在 Python 中被称之为 generator（生成器）

简单地讲，yield 的作用就是把一个函数变成一个 generator，带有 yield 的函数不再是一个普通函数，Python 解释器会将其视为一个 generator，调用 fab(5) 不都会执会执行 fab 函数，而是返回一个 iterable 对象！在 for 循环执行时，每次循环行 fab 函数内部的代码，执行到 yield b 时，fab 函数就返回一个迭代值，下次迭代时，代码从 yield b 的下一条语句继续执行，而函数的本地变量看起来和上次中断执行前是完全一样的，于是函数继续执行，直到再次遇到 yield。看起来就好像一个函数在正常执行的过程中被 yield 中断了数次，每次中断都会通过 yield 返回当前的迭代值

第二周学习笔记
本周主要列举一些学习经验（hexo+github小白的学习之路，不正确的地方望指出）
1：如何更换主题（由默认的Landscape主题更换为我现在的主题next为例）
1）下载next主题
在hexo安装目录下用Git Bash here进入命令框输入以下命令下载next主题
git clone https://github.com/iissnan/hexo-theme-next themes/next
2)运用next主题
进入Hexo文件夹下打开配置文件_config.yml,找到theme字段，将其改为next
3）本地验证主题
hexo目录下，输入hexo s -debug进行调试，
此时本地浏览器输入http://localhost:4000/ 就能预览到该主题的默认效果
4）菜单项的设定
Themes目录下的config.yml配置文件，打开，其中menu节点，将几个注释都给去掉，
本地浏览器刷新看看博客首页菜单的变化，多试几次看看变化，改成自己喜欢的菜单
我是将所有注释都去掉了，代码如下
menu:
home: / || home
about: /about/ || user
tags: /tags/ || tags
categories: /categories/ || th
archives: /archives/ || archive
schedule: /schedule/ || calendar
sitemap: /sitemap.xml || sitemap
commonweal: /404/ || heartbeat
2：主页添加自己的头像（这里我遇到了一些问题）
在Hexo/source目录下新建文件夹uploads,将自己图片放入，在next/config.yml文件中搜索avatar，
